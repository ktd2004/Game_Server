2. 게임 서버 상세 기능
   *. Gateway Server
     클라이언트를 인증한후 인증된 세션을 정보를 Master Server 로 전송 
     Master Server에 세션정보가 등록되면 Channel Server로 세션 정보를 전송
     인증된 클라이언트가 전송한 패킷을 Channel Server 로 전송
   *. Channel Server
     클라이언트가 전송한 패킷을 처리
   *. Master Server
     중복 로그인 여부를 체크

3. 클라이언트 인증 절차
   *. 최초 로그인
      1) 게이트웨이서버에 인증 진행 세션을 등록
      2) 마스터서버에 로그인 요청
      3) 마스터서버에서 로그인 응답 오면
      4) 채널서버에 로그인 요청
      5) 채널서버에서 로그인 응답 오면
      6) 클라이언트에 인증 결과 알림
      7) 예외처리
         마스터서버 등록 실패시 클라이언트를 Kick
         채널서버 등록 실패시 클라이언트 Kick
         마스터서버는 동일 세션 존재하면 기존 세션정보를 새로운 세션 정보로 갱신처리후 중복 로그인 세션을 Kick 요청 

   *. 인증 진행 중인 경우
      1) 게이트웨이서버에 동일 세션이 존재하면 인증 진행중인 세션 찾아서 클라이언트 Kick
      2) 마스터서버에 로그인 요청( 마스터는 기존 세션 갱신 처리후 중복로그인 응답 전송 )
      3) 게이트웨이서버에 새로운 만든 세션을 생성하고 등록 
      4) 마스터서버에서 로그인 요청
      5) 채널서버로 로그인 요청( 채널은 기존 세션을 종료 처리후 새로운 세션을 만들어 중복로그인 응답 전송 )
      6) 클라이언트에 인증 결과 알림

   *. 인증 완료된 경우
      1) 게이트웨이서버에 동일 세션이 존재하면 인증 진행중인 세션 찾아서 클라이언트 Kick
      2) 마스터서버에 로그인 요청( 마스터는 기존 세션 갱신 처리후 중복로그인 응답 전송 )
      3) 게이트웨이서버에 새로운 만든 세션을 생성하고 등록 
      4) 마스터서버에서 로그인 요청
      5) 채널서버로 로그인 요청( 채널은 기존 세션을 종료 처리후 새로운 세션을 만들어 중복로그인 응답 전송 )
      6) 클라이언트에 인증 결과 알림


4. 클라이어트 접속 종료 절차
   *. 인증된 클라이언트
      1) 마스터로 로그오프 요청
      2) 채널로 로그오프 요청
      3) 세션삭제

   *. 인증 진행중인 클라이언트
      1) 마스터로 로그오프 요청
      2) 채널로 로그오프 요청
      3) 세션삭제

   *. 인증 안된 클라이언트
      1) 세션삭제
